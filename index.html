🔍 Логика работы нейросети (простыми словами)

Если вы знакомы с линейной регрессией и деревом решений, нейросеть понять легко — это их эволюция.

Сравнение с линейной регрессией: Регрессия: y = w₁x₁ + w₂x₂ + b — просто взвешенная сумма. Нейросеть делает то же самое, но добавляет функцию активации (например, ReLU). Это позволяет моделировать нелинейные зависимости.

Сравнение с деревом решений: Дерево задаёт жёсткие вопросы: «Если площадь > 50 → влево». Нейросеть создаёт плавные переходы — как если бы тысячи маленьких экспертов голосовали за результат.

Обучение: Модель делает прогноз → вычисляет ошибку → методом обратного распространения корректирует веса → повторяет.

Аналогия:
📊 Регрессия — один эксперт с линейкой.
🌲 Дерево — менеджер с чек-листом.
🧠 Нейросеть — совет директоров, где каждый нейрон вносит взвешенный вклад.

Задача по алгоритму RSA

1. Простые числа: p = 5, q = 7
2. n = p × q = 35
3. φ(n) = (p − 1)(q − 1) = 24
4. Выбрано e = 5 → НОД(5, 24) = 1 ✅
5. Найдено d = 5, так как (5 × 5) mod 24 = 1

🗝️ Ключи:
  Открытый ключ: (n, e) = (35, 5)
  Закрытый ключ: (n, d) = (35, 5)

📩 Сообщение: m = 2
🔒 Шифрование: C = me mod n = 25 mod 35 = 32
🔓 Расшифровка: M = Cd mod n = 325 mod 35 = 2

Из чего состоит доменное имя?

Доменные имена — это удобный способ обращения к серверам в интернете. Пример: blog.service.example.com

⚠️ Используется условный домен example.com — зарезервирован для документации.

Уровни доменной иерархии (справа налево)

1. Корневой домен (.)
Неявная точка в конце любого домена.

2. Домен верхнего уровня (TLD)
Например: .com, .org, .ru.

3. Домен второго уровня (SLD)
Основное имя: example → example.com.

4. Поддомены
service.example.com — сервис
blog.service.example.com — блог внутри сервиса

Визуальная структура

.
└── com
    └── example
        └── service
            └── blog

Зачем это знать?

  • Для настройки DNS и хостинга
  • Для организации микросервисной архитектуры
  • Для повышения безопасности
  • Чтобы лучше понимать, как работает интернет!
